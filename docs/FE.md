## UI Library

The application leverages [Untitled UI](https://untitledui.com/) as its primary UI library. Untitled UI offers a comprehensive set of pre-designed components that help build a consistent and visually appealing user interface.

Unlike other component libraries, Untitled UI is not a pre-built library. Instead, it provides a collection of design tokens, utility classes, and base components that are installed into the project.

The `src/governance-app-frontend/untitledui` directory contains the code generated by Untitled UI. Components are installed in a designated folder, while utilities and styles are added to the `/src` directory. We manually move them into the `untitledui` folder to keep them organized and separate from the rest of the application code.

Components are split into two folders: base and custom. The former contains copied and pasted components from the Untitled documentation, while the latter includes slightly modified components tailored to our use case, such as the `Button`, which integrates with our router's `Link` component.

Additionally, we manually manage an `index.ts` file to export all components without the need for deep imports. This file is not generated by Untitled UI and must be updated whenever new components are added or removed.

Icons are provided by [Lucide](https://lucide.dev/), an open-source icon library. We use the React version of Lucide, which is installed as a dependency in our project.

### Sample steps to add a new Untitled UI component

0. To access PRO (paid) components: `npx untitledui login`.
1. Run `npx untitledui add {{name}}`.
2. Remove `src/governance-app-frontend/src/utils/cx.ts`.
3. Fix imports:
    - `@utils/cx` → `@untitledui/utils/cx`.
    - `@components/untitledui/` → `@untitledui/`.
    - Replace icon files with `"lucide-react"` icons (most names are compatible).
4. Remove the generated icon files, since we are using Lucide instead.
